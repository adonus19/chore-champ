Bootstrap records created

This route only creates the records needed for the first parent-owned household.

  • authAccounts/<uid> for account bootstrap
  • people/<personId> for the canonical parent identity
  • households/<householdId> for the family workspace
  • households/<householdId>/members/<personId> as the owner membership

If Firebase Auth succeeds but Firestore bootstrap fails, this page lets the same signed-in parent retry setup without creating a second account.
